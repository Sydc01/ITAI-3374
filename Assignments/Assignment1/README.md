# Assignment 1 – Development Environments for Edge AI and IoT

## Overview
This assignment explored the tools and environments used to build and deploy AI systems in edge computing and IoT environments.

## Problem
As AI systems move from the cloud to edge devices, developers need tools that support efficient development, deployment, and real-time processing.

## Approach
The assignment analyzed several key technologies used in Edge AI, including:
- Visual Studio Code (VS Code)
- Node.js
- Edge Impulse CLI
- TensorFlow & TensorFlow Lite
- Google Colab
- Generative AI coding tools (e.g., GitHub Copilot)

Each tool was evaluated based on its purpose, functionality, and real-world applications.

## Key Concepts Learned
- Edge AI reduces latency and improves privacy by processing data locally
- Development tools play a major role in how efficiently systems are built
- TensorFlow Lite is critical for deploying models on low-resource devices
- Node.js enables real-time communication in IoT systems
- Generative AI tools can speed up development significantly

## Results / Takeaways
This assignment showed how different tools work together to create a full pipeline—from development to deployment. It highlighted the importance of choosing the right tools depending on system requirements.

## Simple Reflection
This assignment helped me understand how many tools are involved in building real-world AI systems. Before this, I didn’t realize how important the development environment is. It also showed me how AI is moving toward edge devices instead of relying only on the cloud.
